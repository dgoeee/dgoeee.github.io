# 循环轨道

## 目录总览

目录也就是文件夹路径的汇总说明，这些路径是相对根路径的

这里直接用地址作为目录的章节名字，若其有更详细更合适的名字，会在相应的 index.mdx 文件中注明

通常，某个地址的默认呈现页面就是 index.mdx，可以称之为封面

### [/docs/base ](/docs/base/index)

🚧施工中

本目录即在这个路径下保存，具体为 `/docs/base/index.mdx`，也即是 base 区域的封面

同时和纸质书籍的编撰类似，这个路径下还保存着外围信息、前言等文件

也就是说它的意义既是内容的出发点，也是内容的地基

举例来说，考虑到网站这个呈现方式，那么讲述如何构建网页的文章就应当存放在 base 区

我的语言习惯和其他人会有不同，为了避免歧义而写的习惯说明等也应当存放在 base 区

当然诸如版本号、更新日志、引流链接等自然在 base 区

不过这里说明一下版本号这个问题，我的构建中将版本号自动标注在了网页最下方，就不用单独提出了

传统的三段式版本号不适合大量频繁地改动，我在此之上添加了项目构件时间来用作精确的版本分割

### [/docs/book1](/docs/book1/index)

形式上很像一本书，分了不同的章节之类的，所以我命名也是以 book 为系列

如果后期章节的体积变大到足以支撑一本书，那就会独立出来成为 book[n]

这是目前（2023年）的主要施工地

### [/docs/book2](/docs/book2/index)

目前（2023年）的测试区，最初建立的目的是测试构建框架对多路径的支持性

### [/blog/legacy](/blog/legacy)

赛博遗物，原本应当是记录各种思绪的地方，但是目前暂时弃用

因为暂时在 book 中记录这些事情会更加方便，这种形式直观且便于管理

### [/blog/campfire](/blog/campfire)

营火余音，指这个路径下的文件就像野外晚上和队友们围坐在火堆旁的讨论

它们是只言片语，未加整理，不成气候
