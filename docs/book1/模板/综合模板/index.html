综合模板

YAML Front Matter信息就是在markdown文件编辑时最上方的，用来描述该文档信息的定义区，其中image是指在别处显示文章链接时将使用的封面或缩略图

51下午三点成功

markdown的官方格式：​

https://markdown.com.cn/

提示

实际上，本网站支持MDX，向下兼容MD。考虑到此网页的核心目标在于简单地记录，侧重于内容而不是形式，所以首推依旧是MD，在需要的时候仍然可以支持MDX

为了方便文件管理，title尽量用Markdown文件的文件名来决定
title: document template
给搜索引擎看的关键字
keywords: [111,222]
显示文档链接时所用的缩略图或封面
image: img/logo.svg

注意此处的slug是什么意思，注意slug要区分大小写
一个完整的网址是这样的 

<协议>://<主机名>/<路径>/<文件slug>?<查询字符串>#<片段>
比如你有一篇标题很长的文章，可以定义一个简化网址，而不是一长串非常长的标题
slug: mu-ban
未完成状态，也就是只会在开发模式显示此文章，当前不是此状态
draft: false
是否隐藏右侧的文档目录
hide_table_of_contents: false

以下是常驻作者的写法，引用自blog文件夹下的authors.yml

这个是常驻作者的写法
authors: 作者1的名字
authors: [作者1的名字, 作者2的名字]
这个是临时作者的写法
authors:
  - 作者1的名字
  - 作者2的名字
  - name: 临时作者的名字
    title: Docusaurus 1 合作创造者
    url: https://github.com/JoelMarcey
    image_url: https://github.com/JoelMarcey.png
    email: jimarcey@gmail.com
可以在每篇文章里单独覆盖修改常驻作者的信息
authors:
  - key: 作者1的名字
    title: 作者1的新头衔
  - key: 作者2的名字
    name: 作者2的新名字
不显示阅读所需时间
hide_reading_time: true
显示 h2 到 h6 标题，最小2最大6，这个一般用不到，用也是设定2-3，因为我已经更改了默认全局2-5
toc_min_heading_level: 2
toc_max_heading_level: 6
